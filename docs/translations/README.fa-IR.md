کافی است `/graphify` را در دستیار کدنویسی مبتنی بر هوش مصنوعی خود تایپ کنید تا کل پروژه شما — کد، مستندات، PDFها، تصاویر، ویدیوها — را به یک گراف دانش تبدیل کند که می‌توانید به‌جای جستجوی grep در میان فایل‌ها از آن پرس‌وجو کنید.

در Claude Code، Codex، OpenCode، Cursor، Gemini CLI، GitHub Copilot CLI، VS Code Copilot Chat، Aider، Amp، OpenClaw، Factory Droid، Trae، Hermes، Kimi Code، Kiro، Pi و Google Antigravity کار می‌کند.

```
/graphify .
```

همین. سه فایل دریافت می‌کنید:

```
graphify-out/
├── graph.html       در هر مرورگری باز کنید — روی گره‌ها کلیک کنید، فیلتر کنید، جستجو کنید
├── GRAPH_REPORT.md  نکات برجسته: مفاهیم کلیدی، اتصالات شگفت‌انگیز، سوالات پیشنهادی
└── graph.json       گراف کامل — هر زمان بدون بازخوانی فایل‌ها از آن پرس‌وجو کنید
```

برای یک صفحه معماری خوانا با نمودارهای جریان فراخوانی Mermaid، دستور زیر را اجرا کنید:

```bash
graphify export callflow-html
```

## نصب

> **بسته رسمی:** بسته PyPI با نام `graphifyy` (دو y) منتشر می‌شود. سایر بسته‌های `graphify*` در PyPI وابسته به این پروژه نیستند. فرمان CLI همچنان `graphify` است.

**مرحله ۱ — نصب بسته:**

```bash
# توصیه می‌شود (uv به‌طور خودکار graphify را در PATH قرار می‌دهد):
uv tool install graphifyy

# جایگزین‌ها:
pipx install graphifyy
pip install graphifyy  # ممکن است نیاز به تنظیم PATH داشته باشد — توضیح پایین را ببینید
```

**مرحله ۲ — ثبت مهارت (skill) در دستیار هوش مصنوعی:**

```bash
graphify install
```

تمام. دستیار هوش مصنوعی خود را باز کنید و `/graphify .` را تایپ کنید.

برای نصب مهارت دستیار در مخزن جاری به‌جای پروفایل کاربری، از گزینه `--project` استفاده کنید:

```bash
graphify install --project
graphify install --project --platform codex
```

نصب‌های محدود به پروژه فایل‌ها را درون دایرکتوری جاری می‌نویسند، برای مثال
`.claude/skills/graphify/SKILL.md` یا `.agents/skills/graphify/SKILL.md`، و
یک راهنمایی برای `git add` فایل‌های قابل کامیت چاپ می‌کنند.
فرمان‌های مخصوص هر پلتفرم که از نصب پروژه‌ای پشتیبانی می‌کنند، همان پرچم را می‌پذیرند،
برای مثال `graphify claude install --project` یا `graphify codex install --project`.

> **نکته PowerShell:** به‌جای `/graphify .` از `graphify .` استفاده کنید — اسلش ابتدایی در PowerShell به‌عنوان جداکننده مسیر تفسیر می‌شود.

> **خطای `graphify: command not found`؟** از `uv tool install graphifyy` یا `pipx install graphifyy` استفاده کنید — هر دو CLI را به‌طور خودکار در PATH قرار می‌دهند. با `pip` ساده، `~/.local/bin` (لینوکس) یا `~/Library/Python/3.x/bin` (مک) را به PATH اضافه کنید، یا `python -m graphify` را اجرا کنید.

> **تا جای ممکن از `pip install` در مک/ویندوز استفاده نکنید.** مهارت در زمان اجرا Python را از `graphify-out/.graphify_python` تشخیص می‌دهد؛ اگر این مسیر به محیطی اشاره کند که با محیط نصب `pip` متفاوت باشد، خطای `ModuleNotFoundError: No module named 'graphify'` دریافت خواهید کرد. `uv tool install` و `pipx install` بسته را در محیط ایزوله خود نصب می‌کنند و این مشکل کاملاً برطرف می‌شود.

این کار یک فایل پیکربندی کوچک می‌نویسد که به دستیار شما می‌گوید برای سوالات مرتبط با کدبیس از گراف دانش استفاده کند — با اولویت دادن به پرس‌وجوهای محدود مانند `graphify query "<سوال>"` به‌جای خواندن گزارش کامل یا جستجوی متنی خام. در پلتفرم‌هایی که از hookهای حامل payload پشتیبانی می‌کنند (Claude Code, Gemini CLI)، یک hook به‌طور خودکار قبل از فراخوانی‌های ابزاری از نوع جستجو فعال می‌شود و دستیار را به سمت مسیر گراف هدایت می‌کند. در سایر پلتفرم‌ها (Codex, OpenCode, Cursor, ...)، فایل‌های دستورالعمل دائمی (`AGENTS.md`, `.cursor/rules/`, و غیره) همان راهنمایی پرس‌وجو-محور را ارائه می‌دهند. فایل `GRAPH_REPORT.md` همچنان برای مرور کلی معماری در دسترس است.

برای حذف graphify از تمام پلتفرم‌ها به‌طور همزمان: `graphify uninstall` (افزودن `--purge` برای حذف `graphify-out/` نیز). یا از فرمان مخصوص هر پلتفرم استفاده کنید (مثلاً `graphify claude uninstall`).

## آنچه در گزارش وجود دارد

- **گره‌های خدا** — پر اتصال‌ترین مفاهیم در پروژه شما. همه چیز از اینها می‌گذرد.
- **اتصالات شگفت‌انگیز** — پیوندهایی بین چیزهایی که در فایل‌ها یا ماژول‌های مختلف زندگی می‌کنند. بر اساس میزان غیرمنتظره بودن رتبه‌بندی شده‌اند.
- **«چرایی»** — کامنت‌های درون خطی (`# NOTE:`, `# WHY:`, `# HACK:`)، docstringها و منطق طراحی از مستندات به‌عنوان گره‌های جداگانه استخراج و به کدی که توضیح می‌دهند متصل می‌شوند.
- **سوالات پیشنهادی** — ۴ تا ۵ سوال که گراف به‌طور منحصربه‌فرد قادر به پاسخگویی به آنهاست.
- **برچسب‌های اطمینان** — هر رابطه استنتاج شده با یکی از برچسب‌های `EXTRACTED`، `INFERRED` یا `AMBIGUOUS` علامت‌گذاری می‌شود. همیشه می‌دانید چه چیزی پیدا شده و چه چیزی حدس زده شده است.

## فایل‌هایی که پشتیبانی می‌شوند

| نوع | پسوندها |
|------|----------|
| کد (۳۳ زبان) | `.py .ts .js .jsx .tsx .mjs .go .rs .java .c .cpp .h .hpp .rb .cs .kt .scala .php .swift .lua .luau .zig .ps1 .ex .exs .m .mm .jl .vue .svelte .astro .groovy .gradle .dart .v .sv .svh .sql .f .f90 .f95 .f03 .f08 .pas .pp .dpr .dpk .lpr .inc .dfm .lfm .lpk .sh .bash .json .dm .dme .dmi .dmm .dmf .sln .csproj .fsproj .vbproj .razor .cshtml` |
| تنظیمات MCP | `.mcp.json` `mcp.json` `mcp_servers.json` `claude_desktop_config.json` — گره‌های سرور، ارجاعات بسته و نیازمندی‌های متغیر محیطی را استخراج می‌کند |
| مستندات | `.md .mdx .qmd .html .txt .rst .yaml .yml` |
| آفیس | `.docx .xlsx` (نیازمند `pip install graphifyy[office]`) |
| Google Workspace | `.gdoc .gsheet .gslides` (اختیاری؛ نیازمند احراز هویت `gws` و پرچم `--google-workspace`؛ Sheets نیازمند `pip install graphifyy[google]`) |
| PDFها | `.pdf` |
| تصاویر | `.png .jpg .webp .gif` |
| ویدیو / صدا | `.mp4 .mov .mp3 .wav` و موارد دیگر (نیازمند `pip install graphifyy[video]`) |
| YouTube / URLها | هر URL ویدیویی (نیازمند `pip install graphifyy[video]`) |

کد به‌صورت محلی و بدون فراخوانی API (AST از طریق tree-sitter) استخراج می‌شود. بقیه موارد از طریق API مدل دستیار هوش مصنوعی شما پردازش می‌شوند.

فایل‌های `.gdoc`، `.gsheet` و `.gslides` در Google Drive برای دسکتاپ، اشاره‌گرهای میانبر هستند، نه محتوای سند. برای گنجاندن Google Docs، Sheets و Slides اصلی در استخراج بدون رابط کاربری، ابتدا CLI ابزار
[`gws`](https://github.com/googleworkspace/cli) را نصب و احراز هویت کنید، سپس اجرا کنید:

```bash
pip install "graphifyy[google]"  # برای رندر جداول Google Sheets نیاز است
gws auth login -s drive
graphify extract ./docs --google-workspace
```

همچنین می‌توانید `GRAPHIFY_GOOGLE_WORKSPACE=1` تنظیم کنید. Graphify میانبرها را به‌عنوان فایل‌های جانبی Markdown در `graphify-out/converted/` خروجی می‌گیرد، سپس آن فایل‌ها را استخراج می‌کند.

## دستورات رایج

```bash
/graphify .                        # ساخت گراف برای پوشه جاری
/graphify ./docs --update          # استخراج مجدد فقط فایل‌های تغییر کرده
/graphify . --cluster-only         # اجرای مجدد خوشه‌بندی بدون استخراج مجدد
/graphify . --cluster-only --resolution 1.5      # جوامع ریزدانه‌تر
/graphify . --cluster-only --exclude-hubs 99     # حذف ابر-هاب‌های ابزاری از رتبه‌بندی گره‌های خدا
/graphify . --no-viz               # رد کردن HTML، فقط گزارش + JSON
/graphify . --wiki                 # ساخت ویکی Markdown از گراف
graphify export callflow-html      # HTML معماری/جریان فراخوانی Mermaid (در صورت نصب hook، در هر کامیت git به‌طور خودکار بازتولید می‌شود)

/graphify query "چه چیزی auth را به پایگاه داده متصل می‌کند؟"
/graphify path "UserService" "DatabasePool"
/graphify explain "RateLimiter"

/graphify add https://arxiv.org/abs/1706.03762   # دریافت یک مقاله و افزودن آن
/graphify add <youtube-url>                       # رونویسی و افزودن یک ویدیو

graphify hook install              # بازسازی خودکار در هر کامیت git
graphify merge-graphs a.json b.json              # ترکیب دو گراف

graphify prs                       # داشبورد PR: وضعیت CI، وضعیت بازبینی، نگاشت worktree
graphify prs 42                    # بررسی عمیق PR شماره ۴۲ با تأثیر گراف
graphify prs --triage              # هوش مصنوعی صف بازبینی شما را رتبه‌بندی می‌کند (از هر backend پیکربندی شده استفاده می‌کند)
graphify prs --conflicts           # PRهایی که جوامع گرافی مشترک دارند — ریسک ترتیب ادغام
```

## راه‌اندازی تیمی

پوشه `graphify-out/` طوری طراحی شده که در git کامیت شود تا همه اعضای تیم با یک نقشه شروع کنند.

**افزودن‌های توصیه شده به `.gitignore`:**
```
graphify-out/manifest.json    # مبتنی بر mtime، پس از git clone خراب می‌شود
graphify-out/cost.json        # فقط محلی
# graphify-out/cache/         # اختیاری: برای سرعت کامیت کنید، برای کوچک نگه داشتن مخزن رد کنید
```

**گردش کار:**
1. یک نفر `/graphify .` را اجرا کرده و `graphify-out/` را کامیت می‌کند.
2. همه pull می‌کنند — دستیار آنها بلافاصله گراف را می‌خواند.
3. `graphify hook install` را اجرا کنید تا پس از هر کامیت به‌طور خودکار بازسازی شود (فقط AST، بدون هزینه API). این کار همچنین یک درایور ادغام git تنظیم می‌کند تا `graph.json` هرگز با نشانگرهای conflict مواجه نشود — دو توسعه‌دهنده که همزمان کامیت می‌کنند، گراف‌هایشان به‌طور خودکار ادغام می‌شوند.
4. وقتی مستندات یا مقالات تغییر می‌کنند، `/graphify --update` را برای بازخوانی آن گره‌ها اجرا کنید.

## متغیرهای محیطی

این متغیرها فقط برای **استخراج بدون رابط کاربری / CI** (`graphify extract`) نیاز هستند. هنگام اجرا از طریق مهارت `/graphify` درون IDE، API مدل توسط نشست IDE شما تأمین می‌شود — نیازی به کلید اضافی نیست.

| متغیر | مورد استفاده | زمان نیاز |
|---|---|---|
| `ANTHROPIC_API_KEY` | backend کلود (Anthropic) | `--backend claude` |
| `GEMINI_API_KEY` یا `GOOGLE_API_KEY` | backend گوگل Gemini | `--backend gemini` |
| `OPENAI_API_KEY` | OpenAI یا APIهای سازگار با OpenAI | `--backend openai` |
| `DEEPSEEK_API_KEY` | backend دیپ‌سیک | `--backend deepseek` |
| `MOONSHOT_API_KEY` | backend Kimi Code | `--backend kimi` |
| `OLLAMA_BASE_URL` | URL استنتاج محلی Ollama | `--backend ollama` (پیش‌فرض: `http://localhost:11434`) |
| `OLLAMA_MODEL` | نام مدل Ollama | `--backend ollama` (پیش‌فرض: تشخیص خودکار) |
| `GRAPHIFY_OLLAMA_NUM_CTX` | بازنویسی اندازه پنجره کش KV در Ollama | اختیاری — به‌طور پیش‌فرض اندازه‌دهی خودکار |
| `GRAPHIFY_OLLAMA_KEEP_ALIVE` | دقایق نگهداری مدل Ollama در حافظه | اختیاری — `0` برای تخلیه پس از هر قطعه |
| `AWS_*` / `~/.aws/credentials` | AWS Bedrock — زنجیره اعتبارنامه استاندارد | `--backend bedrock` (بدون کلید API، با IAM) |
| `GRAPHIFY_MAX_WORKERS` | تعداد نخ‌های موازی‌سازی AST | اختیاری — پرچم `--max-workers` نیز موجود است |
| `GRAPHIFY_MAX_OUTPUT_TOKENS` | افزایش سقف خروجی برای متون متراکم | اختیاری — مثلاً `32768` برای فایل‌های بزرگ |
| `GRAPHIFY_API_TIMEOUT` | زمان انتظار HTTP به ثانیه (پیش‌فرض: ۶۰۰) | اختیاری — پرچم `--api-timeout` نیز موجود است |
| `GRAPHIFY_FORCE` | بازنویسی اجباری گراف حتی با گره‌های کمتر | اختیاری — پرچم `--force` نیز موجود است |
| `GRAPHIFY_GOOGLE_WORKSPACE` | فعال‌سازی خودکار خروجی Google Workspace | اختیاری — تنظیم روی `1` |
| `GRAPHIFY_TRIAGE_BACKEND` | backend برای `graphify prs --triage` | اختیاری — از کلیدهای موجود تشخیص خودکار |
| `GRAPHIFY_TRIAGE_MODEL` | بازنویسی مدل برای triage | اختیاری — مثلاً `claude-opus-4-7` |

## حریم خصوصی

- **فایل‌های کد** — به‌صورت محلی از طریق tree-sitter پردازش می‌شوند. هیچ چیز از دستگاه شما خارج نمی‌شود.
- **ویدیو / صدا** — به‌صورت محلی با faster-whisper رونویسی می‌شوند. هیچ چیز از دستگاه شما خارج نمی‌شود.
- **مستندات، PDFها، تصاویر** — برای استخراج معنایی به دستیار هوش مصنوعی شما ارسال می‌شوند (از طریق مهارت `/graphify`، با استفاده از هر مدلی که نشست IDE شما اجرا می‌کند). `graphify extract` بدون رابط کاربری به یکی از موارد زیر نیاز دارد: `GEMINI_API_KEY` / `GOOGLE_API_KEY` (Gemini)، `MOONSHOT_API_KEY` (Kimi)، `ANTHROPIC_API_KEY` (Claude)، `OPENAI_API_KEY` (OpenAI)، `DEEPSEEK_API_KEY` (DeepSeek)، یک نمونه در حال اجرای Ollama (`OLLAMA_BASE_URL`)، اعتبارنامه‌های AWS از طریق زنجیره تأمین استاندارد (Bedrock - بدون نیاز به کلید API، با IAM)، یا فایل اجرایی `claude` (Claude Code - بدون نیاز به کلید API، با اشتراک Claude شما). پرچم `--dedup-llm` از همان کلید استفاده می‌کند.
- **محل ذخیره‌سازی داده** — `graphify extract` به‌طور خودکار بر اساس کلید API موجود تشخیص می‌دهد از کدام ارائه‌دهنده استفاده کند (اولویت: Gemini ← Kimi ← Claude ← OpenAI ← DeepSeek ← Bedrock ← Ollama). برای کدهایی که الزامات محل ذخیره‌سازی داده دارند، از `--backend ollama` (کاملاً محلی) استفاده کنید یا یک پرچم صریح `--backend` بدهید. Kimi (`MOONSHOT_API_KEY`) به سرورهای Moonshot AI در چین مسیر می‌دهد.
- بدون تله‌متری، بدون ردیابی مصرف، بدون آنالیز.

## عیب‌یابی

**`graphify: command not found` پس از `pip install graphifyy`**
pip اسکریپت‌ها را در یک پوشه bin کاربری نصب می‌کند که ممکن است در PATH نباشد. راه حل:
- macOS: `~/Library/Python/3.x/bin` را در `~/.zshrc` به PATH اضافه کنید
- Linux: `~/.local/bin` را در `~/.bashrc` به PATH اضافه کنید
- یا از `uv tool install graphifyy` / `pipx install graphifyy` استفاده کنید — هر دو PATH را به‌طور خودکار مدیریت می‌کنند.

**`python -m graphify` کار می‌کند اما فرمان `graphify` کار نمی‌کند**
PATH شل شما شامل پوشه اسکریپت‌های Python نیست. به‌جای `pip` ساده از `uv` یا `pipx` استفاده کنید.

**`/graphify .` در PowerShell باعث خطای «مسیر شناسایی نشد» می‌شود**
PowerShell اسلش ابتدایی را به‌عنوان جداکننده مسیر تفسیر می‌کند. در ویندوز از `graphify .` (بدون اسلش) استفاده کنید.

**گراف پس از `--update` یا بازسازی گره‌های کمتری دارد**
اگر بازآرایی فایل‌ها را حذف کرده باشد، گره‌های قدیمی باقی می‌مانند. پرچم `--force` را بدهید (یا `GRAPHIFY_FORCE=1` تنظیم کنید) تا حتی اگر بازسازی گره‌های کمتری داشته باشد بازنویسی شود.

**گراف گره‌های تکراری برای یک موجودیت دارد (تکرار شبح‌وار)**
این زمانی رخ می‌دهد که استخراج معنایی و AST بر سر قالب شناسه گره توافق نداشته باشند. یک استخراج کامل مجدد برای پاکسازی اجرا کنید:
```bash
graphify extract . --force
```

**Ollama با کمبود VRAM / بیش از حد پنجره context مواجه می‌شود**
پنجره کش KV به‌طور خودکار اندازه‌گیری می‌شود اما ممکن است برای GPU شما بیش از حد بزرگ باشد. آن را کاهش دهید:
```bash
GRAPHIFY_OLLAMA_NUM_CTX=8192 graphify extract ./docs --backend ollama --token-budget 4000
```

**HTML گراف برای باز شدن در مرورگر بیش از حد بزرگ است (>۵۰۰۰ گره)**
تولید HTML را رد کرده و مستقیماً از JSON استفاده کنید:
```bash
graphify cluster-only ./my-project --no-viz
graphify query "..."
```

**`graph.json` پس از کامیت همزمان دو توسعه‌دهنده نشانگرهای conflict دارد**
`graphify hook install` را اجرا کنید — یک درایور ادغام git تنظیم می‌کند که `graph.json` را به‌طور خودکار ادغام می‌کند تا هرگز conflict رخ ندهد.

**استخراج برای مستندات یا PDFها گره‌ها/یال‌های خالی برمی‌گرداند**
مستندات و PDFها نیاز به فراخوانی LLM دارند. بررسی کنید که کلید API شما تنظیم و backend صحیح باشد:
```bash
ANTHROPIC_API_KEY=sk-... graphify extract ./docs --backend claude
```

**هشدار عدم تطابق نسخه مهارت در IDE شما**
نسخه graphify نصب شده با فایل مهارت متفاوت است. به‌روزرسانی کنید:
```bash
uv tool upgrade graphifyy
graphify install  # فایل مهارت را بازنویسی می‌کند
```

## ساخته شده بر پایه graphify — Penpax

[**Penpax**](https://graphifylabs.ai) لایه‌ای همیشه روشن است که بر روی graphify ساخته شده — همان رویکرد گرافی را به کل زندگی کاری شما اعمال می‌کند: جلسات، تاریخچه مرورگر، ایمیل‌ها، فایل‌ها و کد، که به‌طور مداوم در پس‌زمینه به‌روزرسانی می‌شود.

برای افرادی ساخته شده که کارشان در میان صدها مکالمه و سندی پراکنده است که هرگز نمی‌توانند به‌طور کامل بازسازی کنند. بدون ابر، کاملاً روی دستگاه.

**آزمایش رایگان به‌زودی راه‌اندازی می‌شود.** [به لیست انتظار بپیوندید ←](https://graphifylabs.ai)

### راه‌اندازی توسعه

این پروژه از [uv](https://docs.astral.sh/uv/) برای گردش کار توسعه استفاده می‌کند. یک بار نصب کنید، سپس:

```bash
git clone https://github.com/safishamsi/graphify.git
cd graphify
git checkout v8                        # شاخه توسعه فعال

# ایجاد venv پروژه و نصب graphify + تمام افزونه‌ها + گروه dev
# (pytest). uv به‌طور پیش‌فرض گروه وابستگی dev را نصب می‌کند؛ برای رد کردن از --no-dev استفاده کنید.
uv sync --all-extras
```

نصب قابل ویرایش را تأیید کنید:
```bash
uv run graphify --version
uv run python -c "import graphify; print(graphify.__file__)"
```

### اجرای تست‌ها

```bash
uv run pytest tests/ -q                # اجرای مجموعه کامل
uv run pytest tests/test_extract.py -q # یک ماژول
uv run pytest tests/ -q -k "python"    # فیلتر بر اساس نام
```

> نکته macOS: مجموعه تست شامل هر دو فایل نمونه `sample.f90` و `sample.F90` است. اینها روی سیستم‌فایل‌های غیرحساس به بزرگی حروف HFS+ / APFS تداخل دارند. برای تست همزمان هر دو نوع Fortran، روی لینوکس یا در یک کانتینر Docker اجرا کنید.

### گردش کار Git

- توسعه فعال روی شاخه `v8` انجام می‌شود.
- سبک کامیت: `fix: <توضیح>` / `feat: <توضیح>` / `docs: <توضیح>`
- قبل از باز کردن PR، `uv run pytest tests/ -q` را اجرا و تأیید کنید که پاس می‌شود.
- برای هر استخراج‌کننده زبان جدید، یک فایل نمونه به `tests/fixtures/` و تست‌ها را به `tests/test_languages.py` اضافه کنید.

### چه چیزی را مشارکت دهیم

**مثال‌های کارشده** مفیدترین نوع مشارکت هستند. `/graphify` را روی یک مجموعه واقعی اجرا کنید، خروجی را در `worked/{slug}/` ذخیره کنید، یک `review.md` صادقانه بنویسید که بگوید گراف چه چیزهایی را درست و چه چیزهایی را اشتباه گرفته، و یک PR باز کنید.

**باگ‌های استخراج** — یک issue با فایل ورودی، ورودی کش (`graphify-out/cache/`) و آنچه از قلم افتاده یا اشتباه بوده باز کنید.

برای مسئولیت‌های ماژول‌ها و نحوه افزودن یک زبان، [ARCHITECTURE.md](ARCHITECTURE.md) را ببینید.
